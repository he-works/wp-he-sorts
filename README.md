# HE SORTS

> WordPress 관리자 메뉴를 자유롭게 정렬·편집할 수 있는 플러그인

![Version](https://img.shields.io/badge/version-1.0.8-blue) ![WordPress](https://img.shields.io/badge/WordPress-5.0%2B-21759b) ![PHP](https://img.shields.io/badge/PHP-7.4%2B-777bb4) ![License](https://img.shields.io/badge/license-GPL--2.0-green)

---

## 소개

**HE SORTS**는 WordPress 관리자 대시보드의 왼쪽 메뉴를 드래그 앤 드롭으로 자유롭게 재배치할 수 있는 플러그인입니다.  
기본 메뉴 순서가 마음에 들지 않거나, 자주 쓰는 메뉴를 상단에 올리고 싶을 때 간편하게 사용할 수 있습니다.

---

## 주요 기능

- **드래그 앤 드롭 정렬** — 1뎁스(상단 메뉴)부터 3뎁스(중첩 하위 메뉴)까지 자유롭게 순서 변경
- **뎁스 이동** — 항목을 하위로 내리거나(들여쓰기), 상위로 올릴 수(내보내기) 있음
- **메뉴 이름 변경** — 각 항목의 표시 이름을 원하는 대로 수정
- **항목 숨기기** — 필요 없는 메뉴를 삭제하지 않고 숨김 처리
- **커스텀 메뉴 추가** — URL, 아이콘, 권한, 링크 대상을 지정하여 새 메뉴 항목 추가
- **GitHub 자동 업데이트** — WordPress 플러그인 업데이트 창에서 바로 업데이트 가능

## 설치 방법

### 직접 업로드
1. 이 저장소를 ZIP으로 다운로드합니다.
2. WordPress 관리자 → **플러그인 → 새로 추가 → 플러그인 업로드**에서 ZIP 파일을 선택합니다.
3. 설치 후 **활성화**합니다.

### FTP / 파일매니저
1. `plugin-HE-SORTS` 폴더를 `/wp-content/plugins/` 경로에 업로드합니다.
2. WordPress 관리자 → **플러그인**에서 **HE SORTS**를 활성화합니다.

---

## 사용 방법

1. 플러그인 활성화 후 관리자 메뉴 맨 하단의 **HE SORTS** 항목을 클릭합니다.
2. 또는 **플러그인 목록**에서 HE SORTS 옆의 **설정** 링크를 클릭합니다.

### 순서 변경
- 항목 왼쪽의 `⠿` 핸들을 드래그하여 원하는 위치로 이동합니다.

### 뎁스 조정
| 버튼 | 기능 |
|---|---|
| `→` (들여쓰기) | 앞 항목의 하위 메뉴로 이동 (최대 3뎁스) |
| `←` (내보내기) | 한 단계 상위 레벨로 이동 |

### 이름 변경
1. 변경할 항목을 클릭합니다.
2. 우측 **속성 패널**에서 표시 이름을 수정합니다.
3. **적용** 버튼을 누른 후 **저장**합니다.

### 항목 숨기기
- 항목 우측의 👁 버튼을 클릭하면 메뉴에서 숨겨집니다.
- 다시 클릭하면 표시됩니다.

### 커스텀 메뉴 추가
1. 상단 **항목 추가** 버튼을 클릭합니다.
2. 이름, URL, 아이콘, 권한, 링크 대상을 입력합니다.
3. **추가** 버튼을 누르면 트리에 추가됩니다.
4. **저장**합니다.

### 초기화
- **초기화** 버튼을 누르면 모든 설정이 삭제되고 WordPress 기본 메뉴 순서로 복원됩니다.

## 변경 이력

### v1.0.8 (2026-04-23)
- **버그 수정**: 2뎁스·3뎁스로 이동한 메뉴 클릭 시 프론트 404 에러 수정 (서브메뉴 URL `admin.php?page=` 정규화)
- **UX 개선**: 드래그 시 접혀있는 하위 메뉴가 강제로 펼쳐지던 현상 수정 (수동으로 펼친 상태만 유지)

### v1.0.7 (2026-04-23)
- **버그 수정**: 커스텀 메뉴 항목의 `target="_blank"` 설정이 실제 관리자 메뉴에 반영되지 않던 문제 수정 (`admin_head` JS 주입)
- **버그 수정**: 구분선 slug가 WordPress 기본 구분선과 충돌할 수 있던 문제 수정 (`separator-he-sorts-*` 형식으로 변경)
- **보안 강화**: 커스텀 메뉴 아이콘 필드에 이미지 URL 입력 시 올바르게 처리되도록 sanitization 개선
- **안정성**: 드래그 depth 추적을 `document.mousemove` 대신 SortableJS `onMove` 콜백으로 교체 (Firefox 호환성)
- **안정성**: `he_sorts_dfs()` 순환 참조 방지 로직 추가
- **자동 업데이트 수정**: `after_install()` 파일시스템 초기화 개선 및 같은 경로 이동 시 파일 삭제 버그 수정

### v1.0.6 (2026-04-22)
- **버그 수정**: `find_sub_entry()` — 서브메뉴 항목을 다른 상위 메뉴로 이동한 경우 원본 부모에 없으면 전체 `$sub_index`를 순회해 항목을 찾도록 수정
- **버그 수정**: `collect_children()` 주석 및 로직 명확화 — WordPress `$submenu`가 최상위 slug 하나의 키만 지원하므로 depth-3 항목도 동일한 `$sub_key` 아래에 추가되도록 명시
- **버그 수정**: `after_install()` — `$wp_filesystem`이 null인 경우 WP_Error를 반환하도록 안전 처리 추가
- **버그 수정**: `serializeList()` (JS) — `wp_slug`, `parent_id`가 빈 문자열일 때 `null`로 직렬화하도록 수정
- **버그 수정**: `relocateDescendants()` (JS) — 드래그 후 손자(depth-3) 항목의 DOM 순서가 뒤섞이던 문제 수정
- **버그 수정**: `he-sorts-menu.css` — PHP에서 부여하는 커스텀 메뉴 클래스(`he-sorts-custom`)와 CSS 선택자 불일치 수정
- **버그 수정**: README.md 버전 뱃지를 실제 버전에 맞게 수정
